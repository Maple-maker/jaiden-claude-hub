### 2 · STEP 01 — The problem

- Every Claude session starts from zero — no memory of yesterday.
- You burn the first 10 minutes re-explaining your stack and your last decision.
- Death by a thousand context windows.

> **The bottleneck isn't Claude. It's your morning re-briefing ritual.**

```
# every morning
> ok so yesterday I was...
> my stack is... the project...
> and we decided to...
10 min gone. every day.
```

---

### 3 · STEP 02 — What /brand-new-day does

- One command triggers a full intel-gather before Claude writes a word.
- Reads its own memory, yesterday's git commits, your wiki log, your spec files.
- Memory, git, notes — all pulled in parallel.

> **Claude reads its own receipts so you don't have to hand them over.**

```
# /brand-new-day
> /brand-new-day
gather: memory  git  wiki  spec
running 4 sources in parallel...
brief ready.
```

---

### 4 · STEP 03 — What comes out

- Active fronts: every project and where it stands right now.
- Yesterday's work: pulled from real git commits — nothing invented.
- Open threads: unfinished work, ranked by how close to done.

> **It reads like a staff officer briefed the commander. Not a summary — a SITREP.**

```
# AEGIS MORNING BRIEF
ACTIVE FRONTS
  ig-engine     shipping
YESTERDAY (git)
  +macos style, dark guides
OPEN THREADS
  buffer drafts   90% done
```

---

### 5 · STEP 04 — The resume prompt

- The brief closes with a paste-ready prompt for every open thread.
- Copy it. Open a new session. Paste. Working in 30 seconds.
- No re-explaining. Claude already has the context.

> **With great context comes great output. The resume prompt is the whole point.**

```
# resume prompt
> Resume: buffer drafts.
  Context: API wired, video
  step left. Continue from
  the draft-create call.
paste -> working in 30s
```
